= Pre-requisites to deploy the BeeGFS CSI Driver
:hardbreaks:
:nofooter:
:icons: font
:linkattrs:
:imagesdir: ../../media/


[.lead]
Before deploying and configuring the BeeGFS CSI Driver, ensure you have a running Kubernetes cluster, access to a BeeGFS storage system, and the necessary permissions to install cluster-wide resources.

== Verify Kubernetes Setup

Run the following command to check the status of pods in your Kubernetes cluster:

[source,console]
----
kubectl get pods
----

.Expected Output:
[source,shell]
root@node@1:/etc/cni/net.d# kubectl get pods 
No resources found in default namespace. 
----
If no pods are running, you will see an empty list. If pods are present, they should be listed without errors.

== Validate BeeGFS Client and Services

Check that BeeGFS services are running correctly (NOTE: This CSI Driver can work with both beegfs-dkms-client as well as beegfs-client which need to be pre-installed on each Kubernetes node: link:https://doc.beegfs.io/latest/advanced_topics/client_dkms.html[Installation steps^]):

[source,console]
----
beegfs-check-servers
----

=== Prepare Air-Gapped Kubernetes Nodes for Deployment 

For deploying BeeGFS CSI driver in environments where Kubernetes nodes do not have internet access, you must either ensure all necessary images (see deploy/k8s/overlays/default for a complete list) are available on all nodes or ensure they can be pulled from some internal registry.
